## 第3页：用研痛点与需求数据

### 页面标题

为什么要做：辩论教学有价值，但难组织、难反馈、难沉淀

### 页面内容

前期用户调研发现：

| 调研发现 | 数据 / 结论 |
| --- | ---: |
| 学生希望提升表达能力 | 84.7% |
| 学生希望获得赛后具体反馈 | 86.1% |
| 学生愿意尝试 AI 陪练 | 62.5% |
| 学生不愿学习复杂系统 | 31.9% |

三类核心痛点：

1. 学生侧：练习机会少，备赛资料散，赛后不知道如何改进。
2. 教师侧：组织辩论课需要分组、控场、计时、记录和点评，成本高。
3. 教学侧：课堂辩论过程难以沉淀，难以形成长期能力追踪。

需求转化：

| 痛点 | 对应平台能力 |
| --- | --- |
| 学生缺少练习对象 | AI 辩手提供稳定陪练 |
| 学生不知道如何备赛 | 知识库与备赛助手辅助论点准备 |
| 教师组织成本高 | 教师端支持建赛、预约、邀请和签到 |
| 过程记录困难 | WebSocket 房间状态与 Speech 发言记录沉淀 |
| 反馈不具体 | 赛后报告、AI 导师建议和成长趋势 |
| 用户不愿学习复杂系统 | 学生端任务入口前置，教师端使用默认配置 |

### 建议画面

- 左侧放“痛点三角”：学生、教师、教学。
- 右侧放 5 个关键数字。
- 强调 `31.9%` 操作阻力，说明平台必须低门槛。

## 第4页：平台定位与教学闭环

### 页面标题

解决方案：构建课前、课中、课后的 AI 辩论教学闭环

### 页面内容

平台定位：

碳硅之辩是一套面向教学场景的 AI 辩论与表达训练平台，通过 AI 辩手、流程控制、语音记录、知识库备赛和赛后分析，支持学生表达训练与教师形成性评价。

教学闭环：

```text
课前备赛
课程知识库 / 备赛助手 / 论点论据准备

课中辩论
人类学生 vs AI辩手 / WebSocket实时流程 / ASR&TTS语音交互

课后复盘
发言记录 / 评分分析 / 报告导出 / 回放查看

长期成长
能力画像 / 成长趋势 / 班级对比 / 成就激励
```

闭环说明：

| 阶段 | 输入 | 平台处理 | 输出 |
| --- | --- | --- | --- |
| 课前 | 课程文档、辩题、知识点 | 文档解析、向量检索、备赛问答 | 论点、论据、反驳方向 |
| 课中 | 学生发言、AI发言、阶段状态 | 实时通信、流程编排、语音识别与合成 | 完整辩论过程与发言记录 |
| 课后 | 发言文本、音频、评分结果 | AI评估、报告生成、回放整理 | 单场报告、改进建议 |
| 长期 | 多场辩论记录 | 趋势统计、能力画像、班级对比 | 成长曲线、成就激励、教学依据 |

- 每一环放一个小图标：知识库、辩论场、报告、成长曲线。

## 第5页：核心功能总览

### 页面标题

三端协同：学生训练、教师组织、管理员配置

### 页面内容

| 用户端 | 核心功能 | 教学价值 |
| --- | --- | --- |
| 学生端 | 控制台、能力评估、备赛助手、匹配大厅、实时辩论、报告回放、成长分析 | 降低参赛门槛，提升表达训练频次 |
| 教师端 | 班级管理、辩论创建、预约管理、学生邀请、报告查看、教学分析 | 降低组织成本，提升复盘效率 |
| 管理员端 | 用户管理、知识库管理、模型配置、ASR/TTS配置、向量配置、邮件配置 | 支持平台长期运维和多模型扩展 |
| 实时辩论场 | 阶段推进、倒计时、抢麦、AI发言、语音播放、发言记录 | 承载核心人机辩论体验 |

主要功能模块：

学生端：

- 学生个人控制台：展示当前任务、预约辩论、历史记录、成长数据和知识库入口。
- 能力评估：通过五维自评生成能力画像，辅助角色推荐。
- 备赛助手：基于课程知识库提供论点、论据和反驳思路。
- 匹配大厅：支持学生自发创建/加入公开或私密辩论房间。
- 分析中心：展示历史记录、成长趋势、班级对比和成就徽章。

教师端：

- 班级与学生管理：组织教学对象。
- 辩论创建：设置辩题、知识点、时长、轮次和参与学生。
- 预约管理：提前创建辩论赛，发送邀请并管理签到。
- 报告回放：查看学生表现、发言记录和赛后分析。

管理员端：

- 用户、班级、知识库统一管理。
- 大模型、ASR、TTS、向量模型、Coze 智能体和邮件服务配置。

### 建议画面

- 周围三端：学生、教师、管理员。
- 每端只放 3-4 个高频功能，不堆完整菜单。

## 第6页：关键页面：实时辩论与赛后报告

### 页面标题

核心体验：一场辩论如何被记录、分析和复盘

### 页面内容

实时辩论场：

- 支持立论、盘问、自由辩论、总结陈词等结构化阶段。
- WebSocket 实时同步房间状态、当前发言人、倒计时和抢麦状态。
- 学生可文本或语音发言。
- AI 辩手根据辩题、阶段和上下文生成回应。
- TTS 合成 AI 发言，ASR 转写学生语音。

赛后报告：

- 汇总发言内容、时长和阶段表现。
- 生成单场表现评分、优势与改进建议。
- 支持辩论回放、报告查看和导出。
- 长期沉淀为能力趋势和成长记录。

页面重点信息：

| 页面 | 关键组件 | 价值 |
| --- | --- | --- |
| 实时辩论场 | 辩题、当前阶段、倒计时、当前发言人、AI辩手、发言记录、语音控制 | 保证辩论过程按规则推进 |
| 赛后即时分析 | 单场概览、表现评分、优势与改进建议 | 帮助学生快速理解表现 |
| 辩论报告页 | 能力维度、关键发言、详细分析、PDF导出 | 支持学生复盘和教师讲评 |
| 辩论回放页 | 时间线发言记录、音频播放、角色信息 | 还原辩论过程 |
| 成长分析页 | 历史记录、趋势图、班级对比、成就 | 支撑长期能力追踪 |

### 建议画面

- 左侧：实时辩论场截图。
- 右侧：赛后报告 / 成长分析截图。
- 中间用箭头：辩论过程 -> 数据沉淀 -> 报告复盘。

## 第7页：AI辅助教学设计

### 页面标题

AI 角色设计：辅助教师，而不是替代教师

### 页面内容

| AI角色 | 平台能力 | 教学价值 |
| --- | --- | --- |
| AI辩手 | 生成立论、反驳、盘问、总结 | 补足训练对手，提升练习频次 |
| AI评委 | 评分、表现分析、违规检测 | 辅助形成性评价 |
| AI导师 | 生成改进建议和学习反馈 | 支持个性化指导 |
| RAG助教 | 基于课程知识库回答备赛问题 | 提升论据准备效率 |
| 语音助手 | ASR转写、TTS播放 | 支持真实口语训练 |

AI 使用边界：

- AI 提供陪练和初步分析。
- 教师保留最终评价和教学解释权。
- 报告用于辅助复盘，不作为唯一评分依据。

AI 与教学环节对应：

| 教学环节 | AI介入方式 | 平台实现 |
| --- | --- | --- |
| 备赛准备 | 基于知识库回答问题、提供论据方向 | RAGService、DocumentService、pgvector |
| 立论与反驳 | AI辩手根据阶段生成对抗发言 | Debater Agent、Coze/OpenAI兼容接口 |
| 语音训练 | 学生语音转写，AI发言语音播放 | ASR、TTS、音频播放事件回传 |
| 赛后评价 | 分析发言表现，生成改进建议 | Judge Agent、ScoringService |
| 学习指导 | 给出下一步训练方向 | Mentor Agent、AnalyticsService |

### 建议画面

- 中心放“教师与学生”。
- 周围放五类 AI 角色。
- 底部强调“辅助、可控、可解释”。

## 第8页：平台架构与技术栈

### 页面标题

系统实现：前后端分离、实时通信、多智能体与数据沉淀

### 页面内容

架构分层：

```text
前端层
React + TypeScript + Vite
学生端 / 教师端 / 管理员端 / 实时辩论场

接口与业务层
FastAPI + REST API + WebSocket
认证鉴权 / 班级管理 / 辩论任务 / 报告分析

实时编排层
RoomManager + FlowController
阶段推进 / 发言权 / 抢麦 / AI回合触发

AI能力层
Debater Agent / Judge Agent / Mentor Agent
Coze / OpenAI兼容模型 / DashScope ASR&TTS / RAG

数据层
PostgreSQL + pgvector + Redis
用户 / 班级 / 辩论 / 发言 / 评分 / 知识库 / 配置
```

工程质量：

- Docker / Docker Compose 部署。
- SQLAlchemy + Alembic 数据库迁移。
- Vitest / pytest 测试体系。
- JWT 鉴权与角色权限隔离。
- 模型、语音、向量能力后台配置化。

核心技术链路：

1. 实时辩论链路

```text
DebateArena
-> /ws/debate/{room_id}
-> RoomManager
-> FlowController
-> WebSocket广播房间状态
```

2. AI发言链路

```text
辩题 + 当前阶段 + 上下文
-> Debater Agent
-> Coze / OpenAI兼容模型
-> AI发言文本
-> TTS语音播放
-> Speech记录入库
```

3. 赛后报告链路

```text
Speech发言记录 + Score评分结果 + 历史表现
-> ReportService / AnalyticsService
-> 报告页 / PDF导出 / 成长趋势
```

技术栈表：

| 层级 | 技术 |
| --- | --- |
| 前端 | React 18、TypeScript、Vite、Tailwind CSS、Radix UI |
| 后端 | FastAPI、SQLAlchemy、Alembic、Pydantic、JWT |
| 实时通信 | WebSocket、RoomManager、FlowController |
| AI能力 | Debater/Judge/Mentor Agent、Coze、OpenAI兼容模型 |
| 语音能力 | DashScope ASR、TTS、音频播放状态回传 |
| 知识库 | LangChain、RAG、PostgreSQL + pgvector |
| 数据与部署 | PostgreSQL、Redis、Docker、Docker Compose |

### 建议画面

- 五层架构图。
- 右下角放一条调用链：

```text
前端 -> WebSocket/API -> 流程控制 -> Agent/语音/RAG -> 数据库 -> 报告
```

## 第9页：创新点与教学价值

### 页面标题

作品创新：把大模型能力嵌入真实教学流程

### 页面内容

创新点：

1. 人机辩论式教学模式  
   AI 不只是问答工具，而是作为辩手进入完整辩论流程。

2. 多智能体教学协作  
   AI 辩手、AI 评委、AI 导师协同完成训练、评价和反馈。

3. 语音交互与过程记录融合  
   ASR/TTS 支持真实口语训练，发言自动沉淀为复盘数据。

4. 知识库增强备赛  
   学生基于课程资料准备论点，回答可追溯引用来源。

5. 赛后报告与成长分析  
   单次课堂活动转化为可追踪的能力成长数据。

教学价值：

- 对学生：提升表达、逻辑、反驳和 AI 素养。
- 对教师：降低组织成本，获得复盘材料。
- 对课程：支撑 AI 伦理、算法社会影响、数据隐私等议题教学。

创新点归纳：

| 创新方向 | 具体体现 |
| --- | --- |
| 教学模式创新 | 将辩论式教学与 AI 智能体对抗结合 |
| AI应用创新 | 多智能体分工，不止生成文本，还参与陪练、评价和指导 |
| 交互方式创新 | 文本 + 语音 + 实时流程控制，贴近真实辩论场景 |
| 数据沉淀创新 | 将发言、评分、报告、趋势统一沉淀为学习档案 |
| 课程融合创新 | 将 AI伦理、算法影响、数据隐私等议题转化为课堂辩论活动 |

应用价值归纳：

```text
学生：低压力高频训练
教师：低成本组织课堂
课程：高参与度实践教学
学校：AI+教育应用成果
```

### 建议画面

- 上半部分：5 个创新点。
- 下半部分：学生、教师、课程三类价值。
