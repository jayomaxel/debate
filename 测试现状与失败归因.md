## 3. 当前后端失败主因

### 3.1 测试入口仍不统一

归属部分：测试环境 / 运行入口

现象：

- `python -m pytest` 取决于当前全局解释器的依赖状态，当前会直接卡在 `pydantic_settings` 缺失。
- `api\venv\Scripts\python.exe -m pytest` 才是当前能稳定进入完整执行的入口。

影响：

- 如果团队成员直接使用全局 Python，看到的会是“测试根本起不来”。
- 如果使用仓库虚拟环境，看到的才是当前真实失败面。

当前判断：

- 后端测试入口应默认收敛到仓库虚拟环境，而不是依赖外部 Python 环境。

### 3.2 鉴权链路与测试上下文不一致，持续触发大量 `401`

归属部分：中间件 / 路由测试 / 依赖注入

核心现象：

- 多个管理端和学生端路由用例，本应断言 `200`、`400`、`403` 或 `404`，实际统一提前变成 `401 Unauthorized`。
- 这不是零散个例，而是成片出现。

当前受影响最明显的模块：

- `tests/test_admin_kb_router.py`
- `tests/test_admin_router.py`
- `tests/test_admin_router_new_endpoints.py`
- `tests/test_student_kb_router.py`

影响：

- 这类失败直接推高失败数，是当前后端最明显、最集中的共性问题之一。
- 说明测试里构造的用户、token、数据库上下文，与真实鉴权链路接入点仍然没有完全对齐。

### 3.3 测试契约与当前实现发生漂移

归属部分：服务接口 / 模型字段 / 默认值约定

当前已能明确看到的例子：

- `test_admin_response_format.py` 仍在依赖 `AuthService.create_access_token`，但当前实现里该类方法已不存在。
- `test_admin_config_fix.py` 断言默认模型是 `gpt-3.5-turbo`，而当前实现返回的是 `gpt-4`。
- `test_coze_config.py` 仍在断言 `CozeConfig.agent_id`，但当前对象结构已经不再提供这个字段。

影响：

- 这类问题不属于基础设施崩溃，而是“测试还在按旧接口、旧字段、旧默认值验收”。
- 它们会持续制造失败噪音，并误导对业务真实状态的判断。

### 3.4 第三方依赖 API 和 mock 写法没有同步更新

归属部分：测试脚本 / 依赖版本变化 / mock 契约

当前体现明显的几类问题：

- `tests/test_debate_grouping.py` 与 `tests/test_integration.py` 还在使用 `AsyncClient(app=...)`，但当前 `httpx` 版本已不接受这个参数。
- `tests/test_rag_service.py` 中一批用例把 `api_endpoint` mock 成了 `MagicMock`，最终触发 `Invalid type for url`。
- `tests/test_markdown_pdf.py` 与 `tests/test_voice_processor.py` 仍存在异步测试组织方式与当前 pytest 运行方式不匹配的问题。

影响：

- 这类问题说明失败不完全来自业务代码，也包括测试脚本对三方库版本和调用方式的假设已经过期。

### 3.5 UUID / 数据类型假设不一致

归属部分：模型类型 / 测试数据构造

现象：

- `tests/test_student_report_pdf_cache.py` 等用例中出现 `'str' object has no attribute 'hex'`。

这通常意味着：

- 测试里传入的是字符串 ID。
- 但当前 SQLAlchemy / 模型字段路径实际期望的是 UUID 对象或兼容类型。

影响：

- 这类问题规模没有 `401` 连锁大，但属于典型的“测试数据构造方式已经不匹配当前实现”。

## 5. 当前优先级判断

如果目标是尽快把后端全量测试恢复到可用状态，当前建议按下面顺序排：

1. 统一后端测试入口

- 明确默认命令应为 `api\venv\Scripts\python.exe -m pytest`。
- 不再以全局 Python 的结果作为后端健康状态判断依据。

2. 优先清理批量 `401 Unauthorized` 链路

- 这是当前最集中的失败来源。
- 应优先检查鉴权中间件、token 生成方式、测试依赖覆盖和测试数据库会话绑定。

3. 对齐测试与当前实现契约

- 包括 `AuthService` 方法名、`CozeConfig` 字段、默认模型值等。

4. 修复依赖 API 和 mock 过期问题

- 包括 `httpx.AsyncClient` 用法、RAG 相关 mock、异步测试组织方式。

5. 最后处理 UUID / 类型细节问题

- 这类问题局部但明确，适合在主链路清理后逐项收口。
