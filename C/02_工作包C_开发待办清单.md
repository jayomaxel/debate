# 工作包C 开发待办清单

## 1. 什么是接口冻结

接口冻结，指的是先把跨模块协作要用到的请求/响应结构固定下来，在一段时间内不随意改字段名、字段含义和返回层级。

对工作包 C 来说，这次冻结的是：

- `AuthSessionContract`
- `WsTicketContract`
- `MaskedConfigResponse`
- `UploadGuardErrorContract`
- `AuditLogEventContract`

它的重点不是“功能已经全部做完”，而是“先把别人要依赖的接口长相定住”。

## 2. 接口冻结的好处

### 2.1 能并行开发

D 可以先按固定合同接前端，不必等 C 把真实会话、ticket、上传安全全部做完。

### 2.2 降低返工

如果没有冻结，后端今天叫 `expires_in`，明天改成 `ttl`，前端和测试会反复跟着改。

### 2.3 更安全地重构

C1/C2/C3 后续会改很多底层逻辑，但只要输出合同不变，重构风险就小很多。

### 2.4 便于测试和验收

每个合同都可以单独写 mock、单测和联调样例，验收时也能明确判断“字段有没有漂”。

### 2.5 更清楚的 owner 边界

跨工作包协作只通过冻结接口进行，不需要直接改别人文件。

## P0：先完成安全核心链路

### 任务 P0-1：把真实登录/刷新链路切到冻结合同

目标：

- 真实 `/api/auth/login` 和 `/api/auth/refresh` 返回 `AuthSessionContract` 结构。
- 登录态字段稳定，不再让前端依赖旧的散装 token 返回。

建议改动：

- `api/services/auth_service.py`
- `api/routers/auth.py`
- `api/utils/security.py`
- `api/tests/test_auth_session.py`

具体动作：

- 让真实登录返回：
  - `access_token`
  - `access_token_expires_in`
  - `session_id`
  - `user.id`
  - `user.username`
  - `user.role`
  - `refresh_strategy`
  - `requires_reauth`
- 明确旧字段兼容策略：
  - 要么短期保留旧字段并新增合同字段
  - 要么一次性切换并同步修测试

验收：

- 登录接口返回字段和 `AuthSessionContract` 一致。
- 刷新接口返回字段和 `AuthSessionContract` 一致。
- 测试覆盖 teacher / student / administrator。

### 任务 P0-2：完成服务端可吊销会话

目标：

- refresh 不再只是自签 JWT。
- 支持当前设备登出、全设备登出、服务端主动失效。

边界提醒：

- 当前 C owner 文件清单里没有专门的 session model 文件。
- 为了不突破边界，建议优先采用 `Redis session store`，而不是先新建数据库会话表。

建议改动：

- `api/services/auth_service.py`
- `api/utils/security.py`
- `api/middleware/auth_middleware.py`
- `api/routers/auth.py`
- `api/tests/test_auth_session.py`

具体动作：

- 设计 session key：
  - 按 `session_id`
  - 绑定 `user_id`
  - 带过期时间
- refresh 只接受服务端仍有效的 session。
- 新增：
  - 当前设备登出
  - 全设备登出
- access token 校验时，如果业务需要，可增加 `session_id` 关联校验。

验收：

- 被吊销会话不能继续 refresh。
- 当前设备登出后当前 session 失效。
- 全设备登出后该用户所有 session 失效。

### 任务 P0-3：把 WebSocket 接入改成 ticket 模式

目标：

- WebSocket query string 里不再传 access token。
- 统一走 `/ws/{room_id}?ticket=...`。

建议改动：

- `api/routers/auth.py`
- `api/routers/websocket.py`
- `api/middleware/auth_middleware.py`
- `api/tests/test_websocket_ticket.py`

具体动作：

- 在 auth 侧新增真实的 ws ticket 获取接口。
- ticket 里最少要绑定：
  - `room_id`
  - `user_id`
  - `expires_at`
  - `single_use` 或短期有效策略
- 在 websocket 路由里移除 `token` query 认证。
- 改成只校验 ticket。

验收：

- 无 ticket 不能连。
- 过期 ticket 不能连。
- 错 room 的 ticket 不能连。
- query string 不再出现 access token。
