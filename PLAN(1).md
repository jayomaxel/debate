# 学生端第一期完整重构计划（含游客模式、登录拦截、刷新与能力评估规则）

## Summary
- 第一期只重构学生端全部页面与站点入口逻辑；游客模式只开放学生端浏览，不开放教师端和管理员端。
- 第二期再同步教师端和管理员端，沿用同一套品牌语言。
- 视觉方向明确为白底、温和、教育产品化，不做科技感、大屏感、AI 炫技感。
- 站点默认不是登录页，而是公开入口；登录变成一个被触发的认证入口。

## 入口与导航逻辑
- 根路径改为公开入口页，游客默认进入学生端公开壳子。
- 游客可浏览的一级区固定为 `首页 / 比赛 / 备赛 / 成长` 的公开版壳子。
- 游客不可浏览教师端、管理员端；若直接访问对应路径，直接跳登录入口。
- 所有需要认证的卡片、按钮、页面统一接入登录拦截层。
  点击后不保留用户意图，直接跳转登录入口。
- 登录成功后的默认落点固定为学生首页，不做动作恢复、不做回跳继续。
- 右上角未登录状态显示登录入口；已登录状态显示头像区。
- 桌面端右上角设置入口采用“悬停下拉”；移动端提供点击展开的等价交互。
- 下拉菜单只做入口分发，例如：个人资料、账号安全、退出登录、进入设置中心。

## 学生端信息架构
- 一级结构固定为 `首页 / 比赛 / 备赛 / 成长`。
- 首页只负责“下一步行动”和“当前状态”。
  详细解释：
  首页只回答 3 件事：我现在处于什么状态、我下一步该做什么、最近有什么值得我注意。
  因此首页只放主行动卡、当前比赛状态、评估完成状态、最近一场比赛结果、轻量成长摘要。
  首页不再展开资料库列表、不再承载 AI 备赛主界面、不再放复杂设置、不再放完整历史列表。
- 比赛区负责完整参赛主线：加入本场辩论、等待准备、正式辩论、综合赛后页。
- 备赛区负责资料库与 AI 备赛助手。
- 成长区负责历史记录、成长趋势、成就、对比分析。
- 设置中心作为独立页面存在，页面内部采用“左侧导航 + 右侧详情”。
  左侧初始导航包含：个人资料、账号安全、能力评估。
  当能力评估锁定后，左侧移除“能力评估”导航项。

## 登录与能力评估规则
- 首次注册并首次登录成功后，弹出能力评估引导弹窗。
- 弹窗提供两个动作：立即评估、稍后评估。
- 若选择稍后评估：
  - 不阻止进入学生首页
  - 首页主行动区持续提示“完成能力评估”
  - 设置中心也保留相关提醒
  - 未完成前不能加入本场辩论
- 能力评估真正的完成入口在设置中心的“能力评估”页。
- 能力评估锁定触发点固定为“学生完成首场正式辩论后”。
- 一旦完成首场辩论：
  - 能力评估不可再修改
  - 设置中心左侧移除“能力评估”导航
  - 相关能力结果只允许只读展示，不再提供编辑入口

## 比赛流程与正式辩论规则
- “加入课堂”相关文案统一改成“加入本场辩论”。
- 输入邀请码成功后进入单一的“等待与准备页”。
  该页合并当前等待页与对阵确认职责，统一展示辩题、角色、阵容、准备提示、开赛状态。
- 正式辩论开始后，辩论页成为最高优先级全屏业务界面，覆盖站内其它界面。
- 正式辩论期间禁止站内退出：
  - 隐藏全站导航
  - 不提供返回首页、切换一级区、进入备赛/成长/设置的入口
  - 拦截前端路由切换
- 正式辩论期间对浏览器关闭、刷新、后退操作给出离开警告。
- 不强行阻断浏览器本身，但必须明确提示“离开将影响当前辩论进程”。
- 比赛结束后才恢复站内导航，并进入综合赛后页。

## 刷新与登录态保留逻辑
- 刷新原则：页面数据默认自动同步，不依赖用户手动点刷新按钮。
- 登录态在浏览器刷新后优先通过 refresh 流程恢复，不再额外叠加“24 小时完整会话”的前端硬限制。
- 认证时效完全以后端现有 `access_token` / `refresh_token` 机制为准，前端只负责按当前 token 状态做刷新与恢复：
  - 应用初始化或浏览器刷新后，只要本地仍有有效 `refresh_token`，就先静默请求一次 refresh，再恢复登录态与用户信息
  - 请求前若 access token 临近过期，优先静默刷新
  - 收到 401 时做一次 refresh 重试
  - refresh 失败则清空登录态并跳登录入口
- 前端会话字段建议至少包含：
  - `access_token`
  - `refresh_token`
  - `token_expires_at`
  - `user_info`
- 不再引入额外的 `session_started_at`、固定 24 小时窗口或其他独立于后端 token 生命周期之外的本地会话截止规则。
- 自动刷新策略按页面分层：
  - 首页：页面进入时拉取一次；窗口重新聚焦或从后台恢复时自动重拉；页面可见时按固定短周期自动刷新摘要数据
  - 比赛区等待准备页：自动轮询或实时订阅比赛状态，不再依赖用户手动刷新
  - 正式辩论页：以 WebSocket 实时状态为准，不提供站内刷新动作
  - 备赛区：资料列表与会话列表进入页自动加载，焦点恢复时自动同步；AI 对话流按流式返回实时更新
  - 成长区：进入页自动加载，窗口聚焦和可见性恢复时自动补拉
  - 设置中心：进入页自动加载个人资料；保存成功后局部刷新并同步全局用户信息
- 自动刷新必须遵守“仅在页面可见时持续刷新，页面隐藏时暂停高频刷新，恢复可见后立即补拉”。
- 若正式辩论页发生浏览器刷新：
  - 先执行登录态静默刷新；refresh 成功后保留登录态
  - 依据路由中的 `debateId` 重新进入辩论页
  - 自动重连 WebSocket 并恢复当前比赛状态显示

## Public Interfaces / App Structure
- 新增公开入口布局与学生端布局分离：
  - `public layout`
  - `student layout`
  - `settings layout`
  - `debate fullscreen layout`
- 新增统一登录拦截能力：
  - `auth guard for pages`
  - `auth guard for actions`
- 新增会话管理模型：
  - `guest mode state`
  - `protected action redirect`
- 不修改后端业务协议、WebSocket 消息协议、音频传输方式。

## Test Plan
- 游客测试：游客只能浏览学生端公开壳子，不能进入教师端和管理员端。
- 登录拦截测试：点击受保护卡片或按钮时直接跳登录入口，不保留原操作意图。
- 登录态测试：刷新页面后优先执行 refresh 恢复登录态；refresh 失败时清空登录态并回到登录入口。
- 自动刷新测试：首页、等待准备页、成长区、备赛区在聚焦恢复和可见性变化后自动同步数据。
- 辩论页测试：正式辩论开始后隐藏站内导航，禁止站内退出；关闭/刷新/后退时出现离开警告。
- 能力评估测试：首次登录弹引导；可稍后评估；未评估前不可加入本场辩论；首场辩论完成后评估入口被移除且不可修改。
- 赛后测试：比赛结束后进入综合赛后页，优先展示结果、亮点、问题与建议。
- 回归测试：保留现有 WebSocket、音频播放、字幕流转、主持权限等核心辩论能力。

## Assumptions
- 游客模式第一期只做学生端公开浏览，不延伸到教师端和管理员端。
- 登录后统一回到学生首页，不做回跳恢复。
- 能力评估锁定后仍可在别处展示只读能力结果，但不再提供设置页编辑入口。
- 正式辩论“不可退出”解释为：不可站内退出，且浏览器离开前必须警告；不是强行物理禁止关闭浏览器。
