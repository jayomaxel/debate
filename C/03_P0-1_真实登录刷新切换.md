# P0-1 真实登录/刷新切到冻结合同

## 本轮目标

完成 `P0-1`：

- 让真实 `/api/auth/login`
- 让真实 `/api/auth/refresh`

都返回工作包 C 冻结后的会话合同字段，而不是只有旧的 token bundle。

## 修改文件

- `api/services/auth_service.py`
- `api/utils/security.py`
- `api/tests/test_auth_session.py`

## 这轮改了什么

### 1. 真实登录/刷新现在都会返回冻结合同字段

新增并稳定返回：

- `access_token`
- `access_token_expires_in`
- `session_id`
- `user.id`
- `user.username`
- `user.role`
- `refresh_strategy`
- `requires_reauth`

### 2. 保留了旧链路兼容字段

考虑到当前仓库前端和旧测试仍在使用这些字段，本轮没有直接删掉旧返回，而是保留：

- `refresh_token`
- `token_type`
- `expires_in`
- `user.account`
- `user.name`
- `user.email`
- `user.user_type`

所以当前真实接口是“冻结合同字段 + 兼容字段并存”。

### 3. `user` 现在是兼容型结构

`user` 里同时保留两类信息：

- 冻结合同需要的最小身份字段：
  - `id`
  - `username`
  - `role`
- 旧前端和旧逻辑还在使用的详细资料字段：
  - `account`
  - `name`
  - `email`
  - `phone`
  - `student_id`
  - `class_id`
  - `user_type`
  - `avatar`
  - `avatar_url`
  - `avatar_mode`
  - `avatar_default_key`

### 4. token payload 带上了 `session_id`

登录和刷新时，access token / refresh token 的 payload 都带上了 `session_id`。

这一步还不是“服务端可吊销会话”，但它为下一步 `P0-2` 做好了会话标识铺垫。

### 5. token 现在带唯一 `jti`

在 `api/utils/security.py` 里给 access/refresh token 增加了唯一 `jti`。

这样刷新时即使发生在同一秒，新 token 也不会和旧 token 完全一样。

### 6. 登录/刷新文档层现在也对齐了真实返回

这次额外补齐了一个之前审查发现的缺口：

- 真实接口虽然已经返回冻结合同字段
- 但 OpenAPI 文档层还没有同步展示这些字段

本次通过在 C owner 范围内新增 auth 响应 schema，并给 `/api/auth/login`、`/api/auth/refresh` 挂上 `response_model`，让接口文档现在也能看到：

- `session_id`
- `access_token_expires_in`
- `refresh_strategy`
- `requires_reauth`

同时保留兼容字段：

- `refresh_token`
- `token_type`
- `expires_in`

## 代码逻辑说明

### `api/services/auth_service.py`

新增/调整的核心逻辑：

- `_build_access_token_payload`
  - 统一构造 access token payload
- `_build_refresh_token_payload`
  - 统一构造 refresh token payload
- `_build_real_auth_session_payload`
  - 统一把真实登录/刷新结果组织成“合同字段 + 兼容字段”

真实登录和真实刷新都走这层统一拼装，避免两个接口字段再次漂移。

### `api/utils/security.py`

更新了：

- `create_access_token`
- `create_refresh_token`

新增 token 唯一性字段 `jti`，同时补了 `iat`。

### `api/schemas/config.py`

新增了文档层和响应层使用的 auth schema：

- `AuthSessionCompatibleUserResponse`
- `AuthSessionRuntimeResponse`
- `AuthSessionApiResponse`

这几层的作用是：

- 冻结合同字段继续保留
- 兼容字段继续保留
- OpenAPI 展示与真实返回结构一致

### `api/routers/auth.py`

给以下真实路由补上了 `response_model`：

- `POST /api/auth/login`
- `POST /api/auth/refresh`

### `api/tests/test_auth_session.py`

新增了真实路由测试：

- teacher 登录
- student 登录
- administrator 登录
- refresh 后仍保留合同字段
- refresh 后沿用同一个 `session_id`
- OpenAPI 文档层暴露冻结字段和兼容字段

## 当前效果

现在真实 `/api/auth/login` 和 `/api/auth/refresh` 已经不再只是旧的 token bundle。

它们已经具备：

- 冻结合同字段
- 后续会话改造需要的 `session_id`
- 面向现有前端的兼容字段

## 这轮没有做的事

以下内容仍然属于下一轮：

- 服务端可吊销会话
- 当前设备登出
- 全设备登出
- refresh 必须依赖服务端 session 状态

也就是说：

本轮完成的是“真实返回结构切换”，还不是“真实会话治理”。
